# /max-encapsulate — アブストラクション化

サブパッチまたはオブジェクト群を別ファイル（アブストラクション）に分離し、再利用可能にします。

## 手順

1. `max.patcher.snapshot` でパッチ構造を取得
2. `max.object.list` で対象オブジェクトと接続を確認
3. 外部との接続点を分析し、inlet / outlet / 引数を設計
4. アブストラクション用の新規パッチファイルを作成
5. 内部オブジェクトを配置し接続
6. `#1`, `#2` 等の引数プレースホルダを設定
7. 元のパッチでアブストラクション名で呼び出すオブジェクトに置き換え
8. 外部接続を再接続
9. コメントでインターフェース仕様を記述

## 引数: $ARGUMENTS

対象: "$ARGUMENTS"

## アブストラクション設計ガイド

| 要素 | ルール |
|------|--------|
| ファイル名 | 小文字 + ハイフン区切り（例: `my-envelope.maxpat`） |
| 引数 | `#1`, `#2` で外部から設定可能にする |
| デフォルト値 | `#1` が未指定の場合のフォールバックを用意 |
| inlet / outlet | 明確なインターフェースを設計 |
| 保存場所 | 親パッチと同じディレクトリ、または Max サーチパス内 |

## サブパッチとアブストラクションの違い

| 項目 | サブパッチ [p] | アブストラクション |
|------|---------------|------------------|
| 保存場所 | 親パッチ内に埋め込み | 別ファイル（.maxpat） |
| 再利用 | 不可（コピペのみ） | 複数パッチから参照可 |
| 引数 | なし | `#1`, `#2` で可能 |
| 編集の影響 | そのインスタンスのみ | 全参照先に反映 |

## 重要ルール
- ファイル名が Max のオブジェクト名になるため、既存オブジェクトと衝突しない名前にすること
- `#1` 等の引数にはデフォルト値を設けること（`#1` 未指定時のエラー防止）
- アブストラクション内の先頭にコメントで使用方法・引数仕様を記述すること
- `bpatcher` で UI 付きモジュールとして使う場合はプレゼンテーションモードも設定すること
- Max サーチパスに配置するか、親パッチと同じフォルダに保存すること
- poly~ で使う場合は `thispoly~` を含めること
