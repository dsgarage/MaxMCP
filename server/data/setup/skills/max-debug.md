# /max-debug — デバッグ支援

パッチの問題を診断し、修正方法を提案します。

## 手順

1. `max.log.read` でコンソールログを取得し、エラーを確認
2. `max.patcher.snapshot` でパッチ構造を取得
3. `max.object.list` で全オブジェクトと接続状態を確認
4. `max.audio.status` で DSP 状態を確認
5. 問題を診断し、修正方法を提案

## 引数: $ARGUMENTS

問題の症状: "$ARGUMENTS"

## 診断チェックリスト

### 音が出ない場合
- [ ] DSP はオンか（`max.audio.status`）
- [ ] dac~ / ezdac~ は存在するか
- [ ] 信号チェーンに断絶はないか（未接続のインレット/アウトレット）
- [ ] gain~ のレベルは 0 になっていないか
- [ ] オブジェクト引数にエラーはないか

### エラーが出る場合
- [ ] コンソールログのエラーメッセージを確認
- [ ] 型の不一致はないか（Signal ↔ Message の混在）
- [ ] 存在しないオブジェクト名はないか
- [ ] パッケージの未インストールはないか

### 動作が不安定な場合
- [ ] trigger なしで評価順が位置依存になっていないか
- [ ] フィードバックループに clip~ / oneshot~ はあるか
- [ ] metro / delay のタイミングは適切か
- [ ] コールドインレットに先にデータが送られているか

## 出力
- 検出した問題の一覧
- 各問題の原因と修正方法
- 修正を自動適用するかユーザーに確認
