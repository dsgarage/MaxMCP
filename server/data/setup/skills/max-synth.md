# /max-synth — シンセサイザー構築

ユーザーの説明に基づき、Max パッチにシンセサイザーを構築します。

## 手順

1. `max.search.pattern` でユーザーの説明に合うテンプレートを検索
2. `max.patcher.info` で現在のパッチ状態を確認
3. テンプレートに基づき `max.object.create` でオブジェクトを配置（自動レイアウト使用）
4. `max.object.connect` で全接続を実行
5. 必要に応じてコメントを追加
6. `max.audio.toggle("on")` で DSP を有効化

## テンプレート種別

| 種別 | オシレーター | フィルター | 用途 |
|------|------------|-----------|------|
| 減算シンセ | saw~ / rect~ | svf~ / lores~ | 一般的なシンセ |
| FM シンセ | cycle~ × 2 | なし | 金属的・ベル音 |
| 加算シンセ | cycle~ × N | なし | オルガン・倍音制御 |
| ウェーブテーブル | wave~ + buffer~ | svf~ | 複雑な波形 |
| ノイズシンセ | noise~ | svf~ / reson~ | パーカッション・SFX |

## 引数: $ARGUMENTS

ユーザーの説明: "$ARGUMENTS"

上記の説明に最も合うテンプレートを選択し、パッチを構築してください。
テンプレートに完全一致しない場合は、最も近いものをベースにカスタマイズしてください。

## 重要ルール
- 必ず `trigger` オブジェクトで評価順を明示すること
- ezdac~ を最終出力に配置すること
- gain~ または *~ でボリューム制御を入れること（直接 dac~ に繋がない）
- コメントオブジェクトで各セクションにラベルを付けること
