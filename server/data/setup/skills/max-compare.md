# /max-compare — オブジェクト比較

2つの Max オブジェクトの違い・使い分けを詳しく解説します。

## 手順

1. `max.search.object` で両方のオブジェクト情報を取得
2. 必要に応じて WebSearch で公式ドキュメントを参照
3. 機能・パラメータ・パフォーマンスを比較分析
4. 使い分けの指針を提示
5. 典型的な使用シーンごとの推奨を提示

## 引数: $ARGUMENTS

比較対象: "$ARGUMENTS"

## 出力フォーマット

以下の構成で比較してください:

### 1. 基本情報

| 項目 | オブジェクトA | オブジェクトB |
|------|-------------|-------------|
| ドメイン | MSP / Max / Jitter | MSP / Max / Jitter |
| 主な用途 | - | - |
| インレット数 | - | - |
| アウトレット数 | - | - |

### 2. 機能の違い
- 共通点
- 相違点（A にあって B にない機能、その逆）

### 3. パフォーマンス
- CPU 負荷の違い
- レイテンシーの違い

### 4. 使い分けガイド
- こういう場合は A を使う
- こういう場合は B を使う

### 5. コード例
- 同じ目的を達成する場合の、それぞれの構成例

## 重要ルール
- 客観的な比較に徹し、一方を不当に貶めないこと
- 「どちらが良い」ではなく「どの場面でどちらが適切か」を示すこと
- 公式ドキュメントに基づいた正確な情報を提供すること
- 非推奨（deprecated）の場合は明示すること
