# /max-presentation — プレゼンテーション UI 構築

パッチのプレゼンテーションモードで表示する操作 UI を構築します。

## 手順

1. `max.patcher.info` で現在のパッチ状態を確認
2. `max.object.list` で既存の UI オブジェクトを把握
3. 必要な UI オブジェクトを `max.object.create` で配置
4. 各 UI オブジェクトのプレゼンテーションモード属性を有効化
5. プレゼンテーションビューでの位置・サイズを設定
6. ラベル（comment）を配置し、操作内容を明示
7. パネル（panel）で背景・グルーピングを設定
8. パッチのプレゼンテーションモードを有効化

## UI オブジェクト一覧

| オブジェクト | 用途 | 典型的な接続先 |
|------------|------|-------------|
| live.dial | ノブ（連続値） | パラメータ制御 |
| live.slider | スライダー | ボリューム・ミックス |
| live.toggle | ON/OFF | バイパス・DSP |
| live.menu | ドロップダウン | 波形選択・モード切替 |
| live.numbox | 数値入力 | 周波数・BPM |
| live.button | トリガーボタン | ワンショット再生 |
| live.text | テキストボタン | 状態表示付きトグル |
| panel | 背景パネル | グルーピング |
| comment | テキストラベル | セクション名 |
| multislider | 複数スライダー | EQ・スペクトル |

## 引数: $ARGUMENTS

ユーザーの説明: "$ARGUMENTS"

## レイアウトガイド

| 原則 | 説明 |
|------|------|
| グルーピング | 機能ごとに panel で囲む |
| 左→右 | 信号フロー順に配置 |
| 上→下 | 重要度順（メイン操作が上） |
| ラベル必須 | 全ノブ・スライダーにラベルを付ける |
| 色分け | セクションごとに panel の色を変える |
| 余白 | オブジェクト間に適切な余白を確保 |

## 重要ルール
- `live.` 系オブジェクトを優先すること（見た目が統一される）
- 全 UI オブジェクトに `@presentation 1` を設定すること
- `@presentation_rect` で位置・サイズを明示的に指定すること
- panel は UI オブジェクトの背面に配置すること（レイヤー順に注意）
- パッチ全体の `@presentation` 属性を 1 に設定すること
- 操作不要なオブジェクト（内部ロジック）はプレゼンテーションに含めないこと
