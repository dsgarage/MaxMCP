# /max-reactive — リアクティブ映像構築

音響信号を分析し、その結果で Jitter 映像パラメータを駆動するパッチを構築します。

## 手順

1. `max.search.pattern` でリアクティブパターンを検索
2. `max.patcher.info` で現在のパッチ状態を確認
3. 音響入力（adc~ / sfplay~ 等）を配置
4. 分析モジュールを構築（peakamp~ / fzero~ / pfft~ 等）
5. 分析値のスムージング・スケーリングを追加（slide~ / scale 等）
6. Jitter 映像パイプラインを構築
7. 分析値を映像パラメータにマッピング（接続）
8. コメントでセクションラベルを付与

## テンプレート種別

| 分析手法 | MSP オブジェクト | 駆動先例 | 用途 |
|---------|----------------|---------|------|
| 振幅追従 | peakamp~ / average~ | 色・サイズ・アルファ | シンプルなVJ |
| ピッチ検出 | fzero~ / sigmund~ | 色相・位置 | 音程連動 |
| FFT スペクトル | pfft~ / fft~ | テクスチャ・メッシュ変形 | スペクトル可視化 |
| ビート検出 | bonk~ / edge~ | トリガー・シーン切替 | リズム連動 |
| エンベロープ | slide~ + snapshot~ | スムーズなパラメータ | 全般 |
| 帯域分割 | cross~ / svf~ | 低域→赤 高域→青 等 | 周波数帯連動 |

## 引数: $ARGUMENTS

ユーザーの説明: "$ARGUMENTS"

上記の説明に最も合う分析手法と映像パイプラインを組み合わせてパッチを構築してください。

## 重要ルール
- 分析値は必ず `slide~` や `rampsmooth~` でスムージングすること（映像がちらつく）
- `snapshot~` で MSP → Max メッセージに変換する際は適切な間隔（20-50ms）を設定すること
- `scale` でパラメータ範囲を適切にマッピングすること（0.-1. や 0-255 等）
- 音響パスと映像パスの同期には `qmetro` を使うこと
- OpenGL 使用時はコンテキスト名を統一すること
- CPU 負荷を考慮し、不要な高解像度分析は避けること
